# 📌 Project Overview

As part of the **Nautilus DevOps Team's cloud adoption strategy**, this project represents the **first foundational phase** of migrating infrastructure to the **AWS Cloud**.

To minimize operational risk and service disruption, the migration process is executed in **structured and manageable phases**.

**Phase 1 focuses on establishing secure authentication mechanisms** to enable safe access to AWS compute resources, specifically **EC2 instances**.

This is achieved by generating a **secure RSA key pair** used for **SSH authentication**.

---

# 🎯 Objective

The objective of this phase is to create a **secure RSA key pair** that will allow engineers to securely connect to EC2 instances using **SSH key-based authentication** instead of passwords.

# 💻 Method 2: AWS CLI

For **automation and infrastructure consistency**, the key pair can also be created using the **AWS CLI**.

---

## AWS CLI Command

```bash
aws ec2 create-key-pair \
    --key-name devops-kp \
    --key-type rsa \
    --query 'KeyMaterial' \
    --output text > devops-kp.pem
```

---

## What This Command Does

The command above performs the following actions:

1. Creates a new key pair in **AWS EC2** with the name `devops-kp`.
2. Retrieves the **private key material** generated by AWS.
3. Saves the private key locally as `devops-kp.pem`.

This `.pem` file will later be used to **authenticate SSH access to EC2 instances**.

---

## 🔐 Secure the Private Key

To protect the private key from unauthorized access, the file permissions must be restricted so that **only the owner can read the file**.

Run the following command:

```bash
chmod 400 devops-kp.pem
```

### Why This Is Important

* Prevents other users on the system from accessing the private key.
* Ensures compliance with **SSH security requirements**.
* Without proper permissions, SSH may refuse to use the key and display a warning.

---

## 🔗 Connecting to an EC2 Instance

Once the EC2 instance is launched with this key pair, connect using SSH:

```bash
ssh -i devops-kp.pem ec2-user@your-ec2-public-ip
```

### Example

```bash
ssh -i devops-kp.pem ec2-user@54.123.45.67
```

---

## ⚠️ Security Best Practices

* Never commit `.pem` files to GitHub.
* Store private keys in a **secure location**.
* Use **IAM roles and least privilege access**.
* Rotate keys periodically.
* Consider using **AWS Systems Manager Session Manager** for advanced security.
